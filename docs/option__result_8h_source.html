16 // Why so long? Option and Result depend on each other. I don't know of a
17 // way to break the cyclic dependency, primarily because they are templated
18 //
19 // Lifetime notes:
20 // - Every change of state must be followed by a destruction (and construction
21 // if it has a non-null variant)
22 // - The object must be destroyed immediately after the contained value is moved
23 // from it.
24 //
25 // Notes:
26 // - Result and Option are perfect-forwarding types. It is unique to an
27 // interaction. It functions like a std::unique_ptr, as it doesn't allow
28 // implicitly copying its data content. Unless explicitly stated via the
29 // .clone() method.
30 // - We strive to make lifetime paths as visible and predictable as
31 // possible.
